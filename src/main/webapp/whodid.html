Group contract

  • Github

    Rules for working with git on Github

    • Don't work directly on the master branch
    • Work must be pushed onto a relevantly named branch
    • Branches should be deleted after successful tests and merge with master
    • The group should be notified on Discord when something is pushed to master

  • Meetings
    • All group members meet on Discord around 9:00 every day. The exact time is subject to change
    • The meetings at 9:00 should start with a quick review of progress made, as well as plans for the day ahead
    • The last topic of the 9:00 meetings should be whether or not the teamwork between the group members is going well, and if not: How can it be improved?

  • Communication
    • All communication within the group should take place over Discord or Zoom
